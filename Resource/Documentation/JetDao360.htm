  DISTRIBUTION
     

Jet 4 / DAO 3.6 installer (Microsoft Access 2000) - Special thanks to Stefan van As

As for compiling this resource! To distribute an Authorware/Director application that uses DataGrip, you must install Jet/DAO. Proper setup of Jet/DAO is very important. Because Jet/DAO is a shared component that only exists in one location on a machine, all applications using Jet/DAO (including Microsoft Access) may stop working if Jet/DAO setup is not done correctly. Jet/DAO requires a more complex installation procedure than simply copying files into a directory. Your Authorware/Director application cannot have its own "private" copy of Jet/DAO in an application directory.

Jet 4 / DAO 3.6 is required for Authorware/Director applications using DataGrip that need to read and write to Microsoft Access 2000 .mdb database files.

This Jet 4 / DAO 3.6 installer is a regular (wizard-style) installer with a progressbar. If you want to run this installer silently (without the wizard and the progressbar), please use the /s parameter, example:
dao36.exe /s

This installer creates a logfile (dao36.log) in the Windows system directory when the Jet/DAO installation is completed. You can check the dao36.log file in the Windows system directory to check whether the installation is done and whether it was successful.

When a reboot of the system is necessary (most of the time this is not necessary, because the system only needs to be rebooted when certain DLL's were in-use at the time of the installation), this installer will prompt the user with a dialog at the end of the installation. The user can confirm the reboot at this time.

This Jet 4 / DAO 3.6 installer includes DLL in-use detection and DLL version checking. It is up-to-date with the Jet/DAO Service Pack 3 and Microsoft Data Access (MDAC) 2.1 version 2.1.2.4202.3
